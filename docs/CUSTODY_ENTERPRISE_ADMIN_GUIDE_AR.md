# دليل الإدارة — نظام العهد المؤسسي

## الوصول
بوابة كيان ← «مخزون الأصول والعهد». الأدوار: مالك/سوبر/أدمن/مدير/أمين عهدة (`custody_officer`) = لوحة الإدارة.

## الإعدادات والأعلام
الإعدادات ← «مزايا المنصّة المؤسسية»: شغّل/أوقف كل وحدة (للمالك/الأدمن فقط). الوحدات المطفأة مخفية وآمنة.
- التشغيلية مفعّلة افتراضيًا. GPS/التأجير/Zoho/التأمين/Offline/الجوال معطّلة حتى إعدادها.

## المهام الأساسية
- **الأصول**: إضافة/تعديل/أرشفة + صور + تصحيح مخزون + نقل + تايملاين.
- **QR**: تُنشأ تلقائيًا (token آمن). إعادة إصدار (يبطل القديم). طباعة ملصقات (QR + باركود + كود + شعار).
- **الحقائب (Kits)**: تعريف طقم (قطع إلزامية/اختيارية) + snapshot نسخة. الصرف يلتقط لقطة المحتوى.
- **الملحقات**: parent/child لكل أصل (بطارية/شاحن/عدسة… إلزامي/اختياري).
- **العهد**: العهد النشطة + فحص الإرجاع (قبول/تلف/صيانة/مفقود/رفض/جزئي بصورة فحص لكل بند).
- **الحوادث**: مراجعة البلاغات (Hold تلقائي على الأصل) + تحويل لصيانة/تأمين/مسؤولية موظف/إغلاق.
- **الصيانة**: فتح/إغلاق + مورد/تشخيص/تكلفة/فاتورة + فحص ما بعد الصيانة قبل الإتاحة.
- **الجرد/الحجوزات/التقارير/سجل الحركات**.

## التنبيهات
تُشغَّل عبر cron (`/api/cron/custody-alerts` كل ساعة، `CRON_SECRET`): استحقاق/تأخير/تصعيد/ضمانات، بلا تكرار.

## Smoke Test (أدوار: موظف/أمين عهدة/أدمن/مالك/مالي/مستأجر تجريبي)
أنشئ أصلًا → QR/ملصق → حقيبة → مشروع → مسح → صور → توقيع → صرف → GPS → بلاغ حادث → طلب إرجاع (Offline) →
مزامنة → فحص → صيانة → استلام بعد صيانة → عودة للمخزون → تقارير → عقد تأجير → مطالبة تأمين → طلب شراء → Zoho (mock).
احذف بيانات الاختبار بعد التوثيق **دون حذف audit logs**.
