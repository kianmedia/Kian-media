# إغلاق ونشر بوابة التأجير — قائمة تحقّق (Rental Close & Deploy)

> الهدف: تشغيل رفع الصور + عرض العقد على الموقع الحيّ، ثم الدمج إلى main والنشر.
> **الجديد المهم:** رفع الصور وتوقيع العقد صار يعمل بجلسة المستأجر مباشرةً — **يكفي تطبيق
> ملف SQL، دون الحاجة لأي متغيّر بيئة جديد.** (مفتاح الخدمة أصبح احتياطيًا فقط.)

## ما الذي تغيّر في هذه الدفعة
1. **رفع الصور (المشكلة الأولى):** كان يعتمد على مسار خادمي يتطلّب `SUPABASE_SERVICE_ROLE_KEY`
   على Vercel — وغيابه يسبّب «تعذر إتمام العملية». الآن الرفع **مباشر بجلسة المستأجر** إلى
   التخزين (سياسة Storage «rental evidence write v2» تسمح لصاحب الطلب)، ثم الإرفاق عبر
   RPC `custody_rental_finalize_evidence` (ممنوحة للمستخدم). لا يحتاج مفتاح خادم. وإن تعذّر
   الرفع المباشر يسقط تلقائيًا إلى المسار الخادمي القديم.
2. **العقد (المشكلة الثانية):** صار العقد يُعرض عبر **رابط واضح «عرض عقد التأجير كاملًا»**
   يفتح نص العقد القانوني الكامل (مع بيانات الطلب) في نافذة، قبل التوقيع — في شاشة إنشاء
   الطلب وفي شاشة توقيع العقد. ويُخزَّن نص العقد الفعلي مع التوقيع كسجلّ.

## الخطوات (بالترتيب)
### 1) تطبيق SQL — إلزامي (هذا وحده يشغّل رفع الصور)
- خُذ Snapshot/Backup للقاعدة أولًا.
- شغّل على Supabase SQL Editor بالترتيب:
  1. `docs/rental_insurance_production_RUNME.sql`  (الأساس — إن لم يُطبَّق بعد)
  2. `docs/rental_v1_final_production_RUNME.sql`   (الموحّد — يحوي bucket + سياسات Storage + RPCs)
- تحقّق من ناتج الـ VALIDATION في نهاية كل ملف (bucket_mime، evidence_policies، rpcs، grants=anon/authenticated).

### 2) متغيّرات البيئة — اختياري الآن (احتياطي فقط)
- `NEXT_PUBLIC_SUPABASE_URL` و`NEXT_PUBLIC_SUPABASE_ANON_KEY` مضبوطة أصلًا (الموقع يعمل بها).
- `SUPABASE_SERVICE_ROLE_KEY` **لم يعد مطلوبًا لرفع الصور**؛ اضبطه فقط إن رغبت بتفعيل المسار
  الخادمي الاحتياطي (وإشعارات البريد الخادمية إن استُخدمت).

### 3) الدمج والنشر
- ادفع الفرع (GitHub Desktop).
- ادمج فرع التأجير إلى `main` (بعد اختبار سريع على Preview).
- Vercel ينشر main تلقائيًا.

## اختبار قبول سريع (على Preview أو بعد النشر)
1. من بوابة العميل: «＋ طلب تأجير جديد» → أكمل البيانات → أضف معدّة → «متابعة إلى الصور والتوقيع».
2. **ارفع صورة لكل معدّة + صورة إجمالية** — يجب أن تظهر ✓ «أُضيفت صورة» (لا رسالة خطأ).
3. اضغط **«عرض عقد التأجير كاملًا»** — يجب أن يظهر نص العقد الكامل ببيانات طلبك.
4. وقّع + علّم الإقرار → «توقيع وإرسال الطلب» → ينجح الإرسال.
5. سيناريو الأدمن: أنشئ طلبًا للعميل حتى «بانتظار التوقيع» → من حساب العميل: افتح الطلب →
   اضغط رابط العقد (يظهر) → وقّع → ينجح.

## إن ظهر خطأ عند رفع الصور
الرسائل الآن دقيقة وتخبرك بالسبب:
- «خدمة حفظ الصور غير مطبقة في قاعدة البيانات — طبّق ملف التحديث» ⇒ لم يُطبَّق SQL بعد (الخطوة 1).
- «مخزن الصور غير مُهيّأ بعد» ⇒ bucket `rental-evidence` غير موجود ⇒ طبّق `rental_v1_final_production_RUNME.sql`.
- «تعذّر رفع الصورة — تأكد من تطبيق تحديث قاعدة البيانات أو أعد تسجيل الدخول» ⇒ سياسة Storage غير مطبّقة أو الجلسة منتهية.
