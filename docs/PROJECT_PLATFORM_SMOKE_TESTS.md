# مصفوفة اختبارات الدخان — Project Platform Smoke Tests

> تُشغَّل يدويًّا **بعد** تطبيق SQL ونشر Vercel، بحساب طاقميّ وحساب عميل تجريبيّين.
> كل بند: الإجراء ← النتيجة المتوقَّعة. أيّ ✗ = أوقِف واعزل قبل الإعلان عن الجاهزية.

## أ) الأساس ودورة الحياة
1. افتح «منصّة إدارة المشاريع» ← تظهر العدّادات والقائمة من نداء واحد، كل بطاقة فلتر فعليّ.
2. أنشئ مشروعًا (Create Project) بعميل مسجَّل ← ينتقل لصفحته، صفّ `project_core` موجود، الشريط يعمل.
3. انقل المرحلة للأمام ← يطالب بالمتطلّبات الناقصة؛ للخلف/الإغلاق ← يطلب سببًا.
4. شريط التقدّم يعرض قيمة واحدة موحّدة (نفسها للطاقم والعميل).

## ب) المسار السريع (8C)
5. «مشروع سريع» بجانب «إنشاء مشروع» (للإدارة فقط) ← معالج بخطوتين، حقول إلزامية: الاسم + العميل.
6. اختر نوعًا بقالب ← تُنشأ مهام/مخرجات القالب؛ شاشة نجاح صريحة.
7. داخل مشروع «سريع»: تبويب «نظرة سريعة» أوّلًا؛ قائمة إنجاز مشتقّة؛ «غير متاح» ليست «لم يتم».
8. «إدارة متقدمة» يكشف باقي التبويبات (لا تبويب محذوف)؛ «التحويل إلى القياسي» لا يغيّر بيانات.

## ج) القوالب (7A)
9. «من قالب» ← إنشاء مشروع من قالب؛ «حفظ كقالب» على الخادم يلتقط المهام/المخرجات.

## د) الهرمية والبرامج (6A/6B/8A/8B)
10. رقِّ مشروعًا إلى «رئيسي» ← يظهر تبويبا «المشاريع الفرعية» و«إدارة البرنامج».
11. أضف مشاريع فرعية ← ترقيم فريد داخل الأب؛ لا مستوى ثالث (لا Child داخل Child).
12. مخطّط الموجة (8B): «معاينة» ثمّ «تطبيق» ← تُنشأ الوحدات؛ إعادة التطبيق بنفس المفتاح تُبلِّغ «سبق تنفيذها».
13. لوحة البرنامج تُركّب المصادر القائمة (لا عدّ مزدوج للأب والفروع).

## هـ) SLA ومصفوفة التسليم (8D) — على مشروع رئيسي
14. تبويب «الالتزامات والتسليم» يظهر للبرنامج فقط.
15. أنشئ التزام «إجمالي عدد الوحدات» هدف N ← النتيجة مشتقّة، تُعلن المعادلة/العيّنة.
16. التزام دوريّ بلا فترة ← يعرض «غير متاح» (لا صفر مخروق).
17. مصفوفة التسليم: التسليم الفعليّ من سجلّ المراحل؛ «غير موثَّق» لا تاريخ ملفَّق؛ CSV/طباعة يعملان.
18. «بانتظار العميل»: يعرض فقط، لا تذكير تلقائيّ.
19. مركز العمليات: تبويب «التزامات البرامج» يعرض المخروق/المقترب فقط.
20. الإدارة التنفيذية: قسم «التزامات البرامج» معلوماتيّ؛ نسبة غير متاحة تظهر «غير متاح» لا ٠٪.

## و) الإغلاق (5C/6C)
21. مشروع مُسلَّم ← «ابدأ الإغلاق» يعمل؛ حالات الإغلاق التالية لها «تابع الإغلاق».
22. مركز الإغلاق: صناديق الوارد/المحفظة/المعرفة/الأرشيف تعمل.

## ز) العزل (حرِج — بحساب عميل تجريبيّ)
23. العميل لا يرى «منصّة إدارة المشاريع» الطاقمية ولا مركز العمليات ولا SLA طاقميّ.
24. في صفحة مشروعه: يرى ملخّص البرنامج فقط إن كان رئيسيًّا ومُفعَّلًا له (`client_program_view_enabled`).
25. استدعاء مباشر لسطح طاقميّ (RPC) بحساب العميل ← `not authorized`.
26. لا مخاطر/حوكمة/موارد/مالية/التزامات غير معلَنة في أيّ سطح عميل.

## ح) الأداء والأخطاء
27. لوحات كبيرة (مركز العمليات، لوحة البرنامج، مصفوفة SLA) تُحمَّل بنداء واحد لكل لوحة، بلا N+1 محسوس.
28. حالات الخطأ تعرض رسالة عربية + «إعادة المحاولة»؛ لا رسالة PostgREST خام ولا UUID/enum خام.
29. لا تعليق أبديّ (مهلة 20 ثانية على القراءات الثقيلة).
